Navigasi dikembangkan untuk:

ğŸ§  Cara Penyusunan Mindmap / Family Tree

Penyusunan mindmap/family tree di Navigasi mengikuti tahapan berikut:

  1. Inventarisasi NSPK yang relevan

    • Kumpulkan seluruh dokumen peraturan dan SNI yang berhubungan dengan bidang/subbidang yang akan dipetakan.
    • Tentukan sistematika atau klasifikasi pengelompokan yang ingin digunakan (misalnya: berdasarkan alur perencanaan).

  2. Menyusun pengelompokan pasal-pasal peraturan/NSPK

    • Setiap pasal atau isi muatan dari dokumen NSPK selanjutnya dikelompokkan sesuai sistematika/klasifikasi dan sistem penulisan dokumen yang telah ditentukan.
    • Dokumen disusun dengan sistem penulisan berformat Markdown (.md) sebagai format dasar penyusunan. Untuk contoh format .md dapat dilihat di bagian "Contoh Format .md Markdown".

  3. Mengonversi dokumen .md ke tampilan mindmap/family tree (* tahap opsional)

    • Dokumen .md selanjutnya diubah menjadi tampilan visual interaktif mindmap/family tree berformat .html melalui situs markmap.js.org.
    • Cukup lakukan copyâ€“paste isi file .md ke kolom editor di laman tersebut untuk melihat hasil mindmap secara langsung dan unduh file .html -nya.

  4. Mengirimkan dokumen .md dan .html ke situs Navigasi

    • Menyiapkan 2 file berikut:
      • File .md , dari dokumen pemetaan NSPK berformat markdown (wajib)
      • File .html , dari hasil konversi melalui situs markmap (opsional)
    • Keduanya dikirim melalui Form Usulan / Pengajuan di situs Navigasi.
    • Dokumen yang dikirim akan diverifikasi dan ditampilkan sebagai bagian dari basis data publik dan penyajian informasi.

ğŸ§¾ Contoh Format .md Markdown:

Catatan:

  • Heading/bab/sub-bab dibuat dengan menuliskan '#' di depan judul/nama. Jumlah '#' merepresentikan level heading (# = heading level 1; ### = heading level 3).
  • Heading terkecil atau sub-bab maksimal yang dapat dicapai adalah heading hingga level 6 seperti '###### Judul'.
  • Untuk level selanjutnya dapat menggunakan List dengan menuliskan '-' yang merepresentasikan bullet poin yang berbentuk daftar.
  • Pada bagian List, gunakan indentasi (spasi tab) untuk menandakan struktur hierarki sebagai anak/cabang atau masuk ke level selanjutnya.
  • List '-' tidak boleh berada di level yang sama dengan heading '#', karena list yang selevel akan hilang atau tidak terbaca dalam mindmap.
